3.2. Scenario Player module¶

Classes and function to execute test scenario’s

class scenarioplayer.ScenarioPlayer(test_system)¶

Stores and executes test scenario’s

add_socket(a_socket, call_back_function)¶

Add the given socket to the list of sockets to be watched.

Parameters:
  • a_socket (socket or zmq socket) – Socket to add
  • call_back_function (function) – function to register in call_backs

If data arrives on the socket, the given call back function is called with as parameters the socket and a reference to the scenario player.

add_step(when, step)¶

Add the given step to the list of scenario steps. :param when: tells when it is to be executed.

The steps in scenario are executed approximately at the time specified. If the time interval between two steps is large (1 s or more) it will be a close approximation. If the time interval is small (<0.1) the execution can be slightly delayed. :type when: int, float, string :param step: a reference to a function that takes a single parameter, a reference to the scenario player. :type step: function

when

If when is a number, it is interpreted as being a number of seconds.

when can also be a string of the following format: “mm:ss.ss”

For instance:

  • “00:30” means 30 seconds
  • “01:40” means 1 minute 40 seconds
  • “00:00.5” means 0.5 a second

Note that this method behaves differently according to when it is called:

  • If it is called before the scenario is started, when indicates offset from when the scenario is actually started.
  • If it is called after the scenario is started, when indicates the number of seconds from when the method is called
